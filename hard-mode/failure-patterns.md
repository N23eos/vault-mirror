# Hard Mode — Failure Patterns

> **Opt-in module.** Deliberately uncomfortable. Confirm with the user before
> running. This is analysis, not therapy — if the material gets heavy, the
> right move is a human professional, and the report should say so.

## Goal

Dissect the anatomy of this person's failures: not the inventory (module 04
has that), but the mechanism — the repeating sequence by which things go
wrong, the early signals, and the self-protective moves that follow.

## Method

- Take the abandoned/failed undertakings from `Analysis/04-cycles.md` (or
  build the list if 04 wasn't run) and pick the 4–6 best-documented cases.
- For each case, reconstruct the timeline in phases: the spark, the peak, the
  first wobble, the drop, the aftermath story. Quote the notes at each phase.
- Overlay the cases. What repeats? Candidate mechanisms to test: scope
  inflation, motivation vs discipline handoff failure, avoidance after first
  negative feedback, competing new spark, external-validation dependency,
  perfectionist stall. Only claim the ones the timelines actually show.
- Study the aftermath stories specifically: how does the person explain each
  failure to themselves afterward? Does the explanation match the timeline?
  The gap between the two is the self-protective move.
- Identify the earliest reliable warning signal — the note-level tell that
  appears before the wobble is visible.

## Output contract — `Analysis/failure-patterns.md`

```
# Failure Patterns
## Case files          (per case: phase timeline with quotes)
## The mechanism       (the repeating sequence, stated plainly)
## Early warning signals  (what shows up in notes before the drop)
## Aftermath stories   (how failures get explained vs what happened)
## What breaks the pattern  (evidence from the exceptions, if any exist)
```

## Rules reminder

Apply the global rules from `prompts/00-start.md`.

## Quality bar

- Is the mechanism derived from the overlaid cases, not imported from
  self-help literature?
- Does every case file quote actual notes at each phase?
- Does the report end on the exceptions (what breaks the pattern), so it is
  usable and not just an autopsy?
